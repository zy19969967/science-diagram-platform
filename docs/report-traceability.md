## Known limitations

- Security is a single shared gateway token controlled by `GATEWAY_API_TOKEN`; it is suitable for a controlled deployment boundary, not multi-tenant access control.
- Readiness checks are local configuration and filesystem checks. They do not call planner, segmenter, PowerPaint, the local FLUX model, OCR, or browser E2E workflows.
- Report alignment remains staged. Several report-level capabilities are represented by explicit known limitations in `docs/known-issues.md` rather than overclaimed as complete production features.
